# 學院 AI 課程分類圖繪製工具 — 專案規劃（修訂版）

## 一、專案動機

過去的課程分類圖過於平面、缺乏設計感、不美觀,因此希望重新設計一個具備**現代感與未來感**的繪圖工具,同時透過 AI 自動化分類流程,降低人工整理負擔。

本工具定位為**單人單機使用**的純前端 Web 應用,跨裝置使用透過「匯出 JSON / 匯入 JSON」的方式進行遷移。

---

## 二、核心功能

### 1. AI 整合

- 可選擇多款 AI 模型:
  - **OpenAI**
  - **Gemini**
  - **Grok AI**
- API Key 儲存於瀏覽器 IndexedDB,並以 Web Crypto API 加密(由使用者設定主密碼派生金鑰),不外傳
- UI 上標註各家 API 的特性,協助使用者選擇,例如:
  - 免費 / 付費
  - 回應速度
  - 分類品質
  - 額度限制

### 2. 資料上傳與前處理

- 支援多種格式:CSV、Excel、TXT、Markdown 等
- 自動剖析資料、抽取出班名
- 提供學科類別的輸入(例如:「人工智慧(AI)」是其中一種,可接受使用者輸入目前這張「課程分類圖」所要整理班名的學科名稱,目前暫時先作為 AI 相關課程分類整理與顯示給學員觀看瞭解之用,但未來可能會擴大推廣到其他課程領域,所以先暫時做這個預留性質的功能)
- 提供**選填的提示詞欄位**,讓使用者輸入額外資料處理需求
- 自動進行:
  - 資料格式清洗
  - 去除重複
  - 去除語意上不屬於該學科的項目
  - 依使用者需求整理、排序

### 3. AI 分類與標籤

- 經 AI 自動歸類為「分類」與「次分類」
- 自動標示 tag:
  - **A. 對象標籤(Role / Persona)**:全員／主管／非 IT(行政、人資、業務…)／IT(開發、維運、架構)
  - **B. 等級標籤(Level)**:基礎★、進階★★、專業★★★
  - **C. 課程屬性標籤**:善(會用)、用(流程化)、管(治理)、造(創新)
  - **D. 主題標籤(Topic)**:生成式 AI、提示工程、RAG/知識庫、Agent、流程自動化、資料治理、雲端 AI、資安合規…等
  - **E. 形式與評量標籤(Format / Assessment)**:一般課程／工作坊／研討會／證照班…等
- 結果以排列整齊、美觀的方式呈現於畫布

### 4. 分類圖呈現(三層結構)

| 層級 | 內容 | 呈現位置 |
|------|------|---------|
| 第一層 | 類別 ↔ 子類別 | 畫布主視圖 |
| 第二層 | 各分類底下的班名 | 雙擊類別後的 popup |
| 第三層 | 每個班名上的標籤 | 滑鼠 hover 於子類別上方時呈現 |

分類圖的三層結構關係,可以匯出成 Excel 檔,或是 Markdown 格式。

- 標籤形狀為**立體小膠囊**,以顏色區分不同標籤類別
- popup 內直接顯示 tag 彩色膠囊,使用者不需下載 Excel 也能快速檢視
- 同一個標籤類別的字體顏色與背景顏色必須有一致性
- Excel 則作為完整的可解釋性紀錄保存用

### 5. 互動行為

- **雙擊**分類 / 次分類 → 以 popup 在畫面中央顯示該類別下所有班名(含 tag)
- 班名過多時,popup 內顯示捲軸
- **拖曳**分類後,原有連線關係必須維持
- 同一份分類關係可切換不同視覺呈現方式,例如:
  - 樹狀圖
  - 階層圖
  - (3-4 種版型可選)

### 6. 狀態回饋

- 頁面上方以 **toast** 告知使用者處理狀態,例如:
  - 「資料上傳中」
  - 「已完成去除重複與分類」
  - 「分類圖由 AI 繪製中」
- 原則:適度提供必要資訊,避免資訊過載

## 四、儲存、匯入與匯出

### 儲存機制

採用**雙軌儲存**策略,純前端、單機使用,跨裝置透過匯出/匯入 JSON 完成遷移。

#### LocalStorage(輕量、同步、UI 啟動就要讀)

| 項目 | 說明 |
|------|------|
| 主題模式(Light / Dark) | 頁面初始化時立即套用,避免閃爍 |
| 目前選用的版型(樹狀圖／階層圖等) | UI 渲染前需要 |
| 目前選用的配色方案 | UI 渲染前需要 |
| 目前使用中的 AI 模型代號 | 輕量設定 |
| 最近開啟的分類圖 ID | 指向 IndexedDB 中的記錄,用來自動載入上次工作狀態 |

#### IndexedDB(大型、結構化、非同步;建議使用 Dexie.js 包裝)

| 項目 | 說明 |
|------|------|
| **API Keys** | 各家 AI 模型的 Key,以 Web Crypto API(AES-GCM + PBKDF2)加密後儲存 |
| **上傳的原始資料** | CSV/Excel/TXT/MD 解析後的內容 |
| **分類圖資料** | 節點、連線、座標、學科名稱、版型設定等完整 JSON |
| **標籤資料庫** | 五大類標籤(對象/等級/屬性/主題/形式)的所有 tag 定義、顏色 |
| **班名 ↔ 分類 ↔ 標籤** 對應關係 | 三層結構的完整關聯資料 |
| **版本歷史快照** | 對應第五點「版本回復」功能,每個版本一筆記錄 |
| **AI 處理結果快取** | 以「資料指紋 + 模型 + 提示詞」hash 為 key,避免重複呼叫付費 API |

#### 為什麼 API Key 不放 LocalStorage

LocalStorage 任何同源 JS 都能輕易讀取,放 IndexedDB 並搭配 Web Crypto 加密(使用者主密碼派生金鑰)安全性較高。雖然瀏覽器端儲存 API Key 本質上仍有風險,但搭配加密可達到「即使匯出檔外流,沒有主密碼也無法解密 API Key」的保護等級。

### 匯出功能

提供兩種匯出模式:

#### A. 單一分類圖匯出(常用)

一次匯出包含兩個檔案:

1. **JSON 檔**:該分類圖的完整資料(節點、連線、班名、標籤關聯、版本歷史),可在其他裝置匯入繼續編輯
2. **Excel 檔**:可解釋性紀錄
   - 列出每個上傳班名
   - 對應的類別 / 子類別
   - 標上的 tag 清單
   - 用途:作為審查、報告、追溯之用

#### B. 完整環境匯出(跨裝置遷移用)

匯出單一 JSON 檔,內含:

- 所有分類圖資料
- 完整標籤資料庫
- 加密後的 API Keys(對方裝置匯入後需輸入主密碼解密)
- UI 偏好設定

用途:更換裝置、重灌系統、跨瀏覽器使用時的完整搬遷。

### 匯入功能

- 支援匯入先前匯出的單一分類圖 JSON 檔,繼續編輯
- 支援匯入完整環境 JSON 檔,還原所有資料(API Key 需輸入原主密碼解密)
- 匯入時偵測衝突(相同 ID 的分類圖已存在),提供「覆蓋／另存新檔／取消」選項

### Excel 匯出格式

匯出的 Excel 也兼具可解釋性紀錄與跨工具相容性:

| 欄位 | 內容 |
|------|------|
| 班名 | 原始班名 |
| 類別 | 第一層分類 |
| 子類別 | 第二層分類 |
| 對象標籤 | 逗號分隔 |
| 等級標籤 | 逗號分隔 |
| 屬性標籤 | 逗號分隔 |
| 主題標籤 | 逗號分隔 |
| 形式標籤 | 逗號分隔 |

## 五、其他功能需求

- popup 在班名很多時的效能處理採用**虛擬捲軸**
- 「分類歷史紀錄」/「版本回復」功能:
  - 每次重大編輯(AI 重新分類、批次修改、匯入)自動建立版本快照
  - 使用者可手動建立命名版本(例如「2025 Q3 定稿版」)
  - 版本資料存於 IndexedDB,可比較與還原
  - 版本快照預設保留最近 N 筆(可設定),超過自動清理最舊的(命名版本不受清理影響)
- 跨裝置使用提示:UI 中提供「目前資料儲存於本瀏覽器」的明確標示,並引導使用者定期匯出備份

---

## 六、技術棧建議

- **前端框架**:依個人偏好(React / Vue / Svelte 皆可)
- **儲存層封裝**:Dexie.js(IndexedDB 的友善包裝,API 接近 SQL 風格)
- **加密**:瀏覽器原生 Web Crypto API(無需第三方套件)
- **Excel 處理**:SheetJS / ExcelJS
- **檔案解析**:PapaParse(CSV)、SheetJS(Excel)、marked(Markdown)
- **畫布繪圖**:依需求選擇 React Flow / Cytoscape.js / D3.js 等
